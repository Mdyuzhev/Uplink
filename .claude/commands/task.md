Взять задачу в работу.

1. Посмотри список задач: `ls Tasks/backlog/`
2. Если задача указана явно (`/task 003`) — бери её
3. Если не указана — бери задачу с наименьшим номером (приоритет)
4. Прочитай файл задачи полностью
5. Перед началом работы выведи план:
   ```
   Задача: [номер] [название]
   План:
   1. ...
   2. ...
   3. ...
   Ожидаемый результат: ...
   ```
6. Выполни задачу по шагам
7. После каждого значимого шага — запусти тесты
8. По завершении — перемести файл: `mv Tasks/backlog/NNN_*.md Tasks/done/`

## Правила

- Следуй стилю кода из CLAUDE.md
- TypeScript strict, ESLint чистый
- Каждый модуль должен иметь тест
- Проверяй что существующие тесты не сломались
- Коммить после завершения задачи

## Префиксы коммитов

`[ext]` расширение, `[matrix]` Matrix-интеграция, `[livekit]` LiveKit-интеграция, `[webview]` React UI, `[infra]` Docker/серверы, `[docs]` документация, `[test]` тесты, `[fix]` баг, `[refactor]` рефакторинг

## Формат коммита

```
[prefix] Краткое описание на русском

- Что сделано
- Какие файлы затронуты
```

Если задача непонятна — не додумывай, спроси.
